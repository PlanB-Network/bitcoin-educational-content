## 设置Trezor

Trezor附带了自己的微型USB线。确保你使用它而不是随便一个旧线缆。一些USB线缆仅供电源使用。这个既可以传输数据也可以供电。如果你使用带有手机充电的USB线缆，设备可能无法连接。

将其连接到你的电脑，设备将开机。你会收到一条消息说“前往Trezor.io/start”。做到这一点，并将Trezor Suite下载到你的电脑上。

## 观察钱包

在之前的文章中，我解释了如何下载和验证Sparrow Bitcoin Wallet，以及如何将其连接到您自己的节点或公共节点。您可以按照这些指南操作：

- 安装Bitcoin Core
- 安装Sparrow Bitcoin Wallet
- 将Sparrow Bitcoin Wallet连接到Bitcoin Core

使用Sparrow Bitcoin Wallet的另一种选择是Electrum桌面钱包，但我将继续解释Sparrow Bitcoin Wallet，因为我认为它对大多数人来说是最好的选择。高级用户可能喜欢使用Electrum作为替代方案（参见我的指南）。

我们现在将启动Sparrow并连接Trezor（带有种子短语但现在带有密码短语）。这个钱包从未暴露给Trezor Suite，因为它将在我们将设备连接到Trezor Suite之后创建。确保您永远不要再将其连接到Trezor Suite，以免暴露您的新钱包。（您可以连接没有密码短语的钱包，因为那可以是您的诱饵钱包）。

接收

要接收一些比特币，转到左侧的地址标签页并选择一个地址来接收。只需右键点击您想要的地址，并选择“复制地址”。然后前往您的交易所，将钱从那里发送过来并粘贴它。或者您也可以给客户这个地址，他们可以用它来支付给您。

当您第一次使用钱包时，您应该接收一小笔金额，练习将其发送到另一个地址，无论是在钱包内还是返回到交易所，以证明钱包按预期工作。

一旦您做到这一点，您必须备份您写下的单词。一份副本是不够的。至少有两份纸质副本（金属的更好），并将它们保存在两个不同的、安全的地方。这减少了自然灾害一次性摧毁您的硬件钱包和纸质备份的风险。参见“使用比特币硬件钱包”以获得完整讨论。

## 发送

![image](assets/27.webp)

进行支付时，您需要在“支付至”字段中粘贴您要支付的地址。实际上您不能留下标签为空，它只是为了您自己的钱包记录，但Sparrow不允许这样做 - 只需输入一些内容（只有您会看到它）。输入金额，您还可以手动调整您想要的费用。

除非HWW连接，否则钱包无法签署交易。这是HWW的工作 - 接收交易，签名，并返回签名后的交易。确保当您在设备上签名时，您视觉上检查您支付的地址在设备和计算机屏幕上是相同的，以及您收到的发票（例如，您可能收到了一封电子邮件，要求支付某个地址）。

还要注意，如果您选择使用的硬币大于支付金额，那么剩余的部分将被发送回您钱包的找零地址之一。有些人不知道这一点，并在公共区块链上查看他们的交易，以为有些比特币被发送到了攻击者的地址，但实际上，那是他们自己的找零地址。

## 附录A - 验证软件下载

![image](assets/28 .webp)

下载签名文件（一个文本文件）和签名密钥（一个文本文件），并记下文件名及您下载文件的位置。

接下来，对于Mac，您需要下载GPG Suite（参见此处的说明）。

对于Windows，您需要GPG4win（参见此处的说明）。

对于Linux，GPG已经是每个包的一部分。如果您还没有安装，可以通过命令安装：sudo apt-get install gpg

接下来，对于Mac/Windows或Linux，打开终端，并输入命令：

```bash
gpg –import XXXXXXXXXX
```

其中XXXXXXXXXX是签名密钥文件的完整路径（完整路径意味着文件所在的目录和文件名）。您应该看到一个成功导入密钥的确认信息。

然后

```bash
gpg –verify ZZZZZZZZZZ WWWWWWWWWW
```

其中ZZZZZZZZZZ是签名文件的完整路径，WWWWWWWWWW是您下载的Trezor Suite程序的完整路径。

您应该在输出中的某处看到一条消息“来自SatoshiLabs的良好签名”。底部有一个警告，可以忽略。
